Options and choices for developing programs on the fx-9750GIII


Developing programs in this calculator can be done in a few different ways. The fx-9750GIII comes with Casio Basic as its "main language", native to most, if not all, Casio calculators, and MicroPython. It can also be programmed in Assembly and C, and possibly even other languages.

MicroPython

MicroPython being included is a great advantage, as it can also run on other computers running Python. It means the programs written in it can be ported easily to, and from the calculator. The version of MicroPython that has been adapted to run on this calculator is slightly different than the Python you will find running on other machines, but it is likely that program ports would require minimal changes.

According to the manual, some special functions restricted to these calculators do exist. One such example is the casioplot module, which includes functions for drawing pixels and character strings. Moreover, the PYTHON mode lacks some modules and libraries that would be otherwise included in MicroPython - The manual specifies the existence of the math, random, and casioplot modules in the fx-9750GIII (software version 3.50). I do not know if it is possible to transfer new libraries to the PYTHON mode through the port included in the calculator.

Another possible advantage of using Python to develop programs in this calculator, is that there are more online resources dedicated to it, and performing the editing on a computer is more trivial than editing Casio BASIC (which uses certain specific symbols and codes for commands), requiring no SDK unless you use functions from the casioplot module, in which case testing programs on a computer becomes more complicated, but perhaps not impossible (see the external links section).

Casio BASIC

Casio BASIC is the language used in the PRGM mode, and works well enough for writing small programs (say, for executing simple physics formulae). It is a bit more of a hassle to write and run, as well as debug, since there are no official tools to run Casio BASIC outside of a calculator that I'm aware of, and editing software using the calculator is mostly not comfortable or ideal. Still, unofficial resources do exist (see the External Links section), although I have not tested them as of this time of writing.

As for learning Casio BASIC, there is a section on programming on the manual for the calculator, and there are websites that teach some of the basics, if not most of it.

Assembly/C/C++/etc

Like all other computers, the calculator must have a processor unit, and must run some sort of machine code: this means it is possible to program it in some sort of assembly language, and by extension, C and C++ (and all other languages that could produce machine code). This, however, is only theoretical, and in practice, having a working assembler and compilers depends heavily on the processor architecture being well documented. Obviously, community efforts could, and in this case, have managed to produce working SDKs for developing programs for the fx9750-GIII and other calculators.

Writing programs in this way is probably the most complicated way of approaching programming on a fx9750-GIII calculator, but it is possible. It may have some benefits in doing so, such as being able to better optimize specific programs, since access to the hardware is more direct. Still, this is probably not good for the faint of heart!
